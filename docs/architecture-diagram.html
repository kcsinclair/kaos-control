kaos-control Architecture

Single-binary lifecycle manager — Go server + embedded Vue 3 SPA, drives Claude Code / Codex / Gemini / Ollama agents over a git-backed markdown corpus.

Backend stack (Go 1.25)

  • • net/http + go-chi/chi v5 router
  • • coder/websocket — live event hub
  • • modernc.org/sqlite — pure-Go index cache
  • • go-git/go-git v5 — branches, history, diffs
  • • fsnotify — debounced filesystem watcher
  • • goldmark + frontmatter — artifact parsing
  • • yaml.v3 — config + frontmatter
  • • argon2id + sessions — auth.Store
  • • embed.FS — SPA bundled into binary

Frontend stack

  • • Vue 3.5 SFCs + TypeScript + Pinia
  • • Vite 5 + vue-tsc build → web/dist
  • • Vue Router 4 — Dashboard, Kanban, Map…
  • • 3D-Force-Graph + three.js — 3D graph
  • • Cytoscape.js + fcose / dagre — 2D graph
  • • CodeMirror 6 — markdown + yaml editor
  • • markdown-it · ECharts · lucide-vue
  • • Native WebSocket — live optimistic UI
  • • Vitest + jsdom — unit tests

Lifecycle domain model

  • • Artifact types: idea, plan-be/fe/test, test, defect, release, sprint, prototype
  • • Status: raw → draft → clarifying → planning → in-development → in-qa → done/approved/rejected
  • • Lineage: shared slug, monotonic index across stages, parent: in frontmatter
  • • Roles: analyst, backend/frontend/test-developer, qa, product-owner, reviewer, approver
  • • 6 agents configured in lifecycle/config.yaml, each with scoped allowed_write_paths
  • • required_plans gate planning → in-development transitions
  • • Disk is authoritative — SQLite is a rebuildable cache

Event bus (WebSocket)

  • • /api/ws — per-project hub, fan-out to all connected clients
  • • artifact.indexed, file.changed — fsnotify-driven
  • • agent.started / progress / finished / failed — streamed driver output
  • • queue.* — rate-limit pause/resume, job state
  • • lock.* — lineage lock heartbeat + release
  • • devops.* — pipeline run logs (ansi-stripped)
  • • Origin patterns derived from public_host config

Security & sandboxing

  • • argon2id password hashing + server-side sessions
  • • API tokens via kaos-control auth subcommand
  • • filepath-securejoin — path-traversal-safe FS access
  • • PermissionHook intercepts Claude Code PreToolUse, denying writes outside the agent's allowed_write_paths
  • • Policy/precheck enforces role + agent scoping before queueing
  • • WebSocket OriginPatterns restrict cross-origin upgrades

Agent integrations

  • • Claude Code CLI — primary driver, hook-mediated
  • • Claude Mediated — Claude as conductor over tool-only runs
  • • OpenAI Codex CLI — stdin/stdout protocol driver
  • • Google Gemini CLI — subprocess driver
  • • Ollama HTTP API — local-LLM cluster fan-out
  • • Streaming progress events surfaced through the hub
  • • Queue handles rate-limit detection with fallback pause + resume grace
