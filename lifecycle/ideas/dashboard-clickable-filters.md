# Dashboard Items as Clickable Filters

All summary counts and status indicators on the dashboard should be interactive. Clicking a count (e.g. blocked, in-review, in-development) should navigate the user to the artefacts list view with the appropriate filter pre-applied, so they immediately see the relevant artefacts without manual filtering.

This applies to all widgets: the Summary Counts widget, Status Distribution widget, and any other dashboard elements that represent a subset of artefacts. The filtered view should reflect exactly the same criteria implied by the clicked item.

The goal is to reduce friction when triaging — the dashboard becomes a launchpad for action rather than a passive summary.
