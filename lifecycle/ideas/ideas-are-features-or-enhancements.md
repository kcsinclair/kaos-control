# ideas are features or enhancements

relates to: [[ideas/tech-writer-agent]]

Pivoting this idea to be about:
* Ideas
* Enhancements
* Defects

An idea may or may not be an epic, feature or enhancement, at the time you have the idea you don't know.

You do know when something you are requesting is an enhancement or defect.

When raising an enhancement or defect you should know which lineage you are raising it against.

Buttons to raise in this order on the artefact list and board:
1. New Idea
2. New Enhancement
3. New Defect

## Ideas
An idea may be a new feature or an enhancement, and should be labeled as best as can be determined, this can be updated later.  This is working well now.

## Enhancements
An enhancement is an idea artefact with a label of enhancement and the lineage matched to the correct existing lineage.

## Defects
Defects are working well already.

## Technical Features
Lineage represents the technical features, we might want enhancement in the future to rename lineages, etc

The Lineage is useful to the developers and agents, but not as much to the end user, features will be the marketed/documented lineages.

## Features
Features are the documentation and marketing artefact, the tech writer should identify features in the system and document around those features, the documentation could include reference to the lineage if that makes sense.

A recognised feature may not be directly related to a single lineage, which is OK, one lineage might be several features.
